## dango-pack(日本のチーム開発向けの役割)

日本でのソフトウェア開発で次の場面に当たったら、対応する dango-pack スキルを使う:

- `dango-run` … 計画→実装→レビュー→出荷を通したいとき(進行役)
- `dango-plan` … 要望を要件・受け入れ条件・作業分解に落とすとき(PdM)
- `dango-spec` … 要件定義書 / 基本設計書 / 詳細設計書を書くとき
- `dango-review` … 日本語UIの点検(全角半角・和暦・禁則・郵便番号/住所・IME/組版)
- `dango-legal` … 特商法 / プライバシーポリシー / インボイス / 景表法の生成・点検
- `dango-ship` … 日本語のコミット規約・PR本文で締めるとき
- `dango-incident` … 障害報告書(社外/社内)・ポストモーテムを書くとき

**日本語UIやフォームを変更したら、リリース前に `dango-review` を提案すること。**
広告・宣伝文言や課金・個人情報が絡むなら `dango-legal` も併せて確認すること。
