# dango-pack — Claude Code 向けの運用指示

このリポジトリは Claude Code / Codex に「日本のチーム開発向けの役割(スキル)」を
与える OSS です。**ユーザーがこのディレクトリで自然言語で頼んだら、対応する
セットアップ用コマンドをあなた(Claude Code)が実行してください。**
ユーザーが npm / node コマンドを覚えていなくても済むようにするのが目的です。

## インストール / セットアップ

ユーザーが「**インストールして**」「セットアップして」「導入して」「使えるようにして」
等と言ったら、リポジトリ直下で次を実行する:

```bash
node bin/install.mjs
```

- これは `skills/` を `~/.claude/skills/` にシンボリックリンクし、役割ガイドを
  `~/.claude/CLAUDE.md` に冪等に注入する(依存ゼロ・ビルド不要)。
- 実行後、`linked: <役割名> -> ...` が役割の数だけ出れば成功。
- 完了したら「Claude Code を再読込(再起動 or 新規セッション)してください」と伝える。
- 失敗時は `node --version`(18 以上か)と `~/.claude/skills/` の書き込み権限を確認する。

## 更新

「**更新して**」「最新にして」等と言われたら:

```bash
git pull
```

スキルはシンボリックリンクなので、`git pull` だけで反映される(再インストール不要)。
リンクが外れている疑いがあれば `node bin/install.mjs` を再実行してよい。
ただし `~/.claude/CLAUDE.md` に注入した役割ガイド(提案トリガ等)を最新にするには、
`git pull` 後に `node bin/install.mjs` を再実行する(マーカー間を冪等に差し替える)。

## アンインストール

「**アンインストールして**」「外して」「無効化して」等と言われたら:

```bash
node bin/uninstall.mjs
```

`~/.claude/skills/` の dango-pack リンクと、注入した CLAUDE.md セクションだけを外す。
本体ディレクトリと成果物(`~/.dango-pack/`)は残す旨を伝える。

## 検証(任意)

スキル定義の妥当性を確かめたいときは `node bin/validate.mjs`。

## 注意

- これらは破壊的ではないが、`install` / `uninstall` は `~/.claude/` を書き換える。
  **実行前に「`node bin/install.mjs` を実行します」と一言告げてから**走らせること。
- このリポジトリ自体のコミット / push / PR 作成は、別途ユーザーの許可を得てから行う。
