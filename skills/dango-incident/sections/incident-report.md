# 生成物: 障害報告書(社外/社内向け)

顧客・経営・関係部署に出す公式記録。日本の現場では「発生〜復旧の時系列」「影響範囲」
「原因」「再発防止策」を明確に分けて書くのが定石。

## 雛形

```
# 障害報告書

## 改訂履歴
| 版 | 日付 | 作成者 | 変更内容 |
|---|---|---|---|
| 1.0 | {{日付}} | {{氏名}} | 初版 |

## 1. 概要
{{何が起きたかを2〜3行で}}

## 2. 発生状況
| 項目 | 内容 |
|---|---|
| 発生日時 | {{YYYY/MM/DD HH:MM}} |
| 検知日時 | {{ }} |
| 復旧日時 | {{ }} |
| 障害時間 | {{ }} |

## 3. 影響範囲
- 対象サービス/機能: {{ }}
- 影響を受けた顧客/件数: {{ }}
- 業務・金銭への影響: {{ }}

## 4. 原因
- 直接原因: {{何が直接の引き金か}}
- 根本原因: {{なぜそれが起きたか。仕組み・プロセスの欠陥}}

## 5. 対応経過(時系列)
| 時刻 | 対応内容 |
|---|---|
| {{HH:MM}} | {{検知・調査・暫定対応・復旧}} |

## 6. 恒久対策・再発防止策
| 対策 | 担当 | 期限 |
|---|---|---|
| {{ }} | {{ }} | {{ }} |

## 7. お詫び(社外向けの場合)
{{誠実な謝罪。誇張も矮小化もしない}}
```

## 原則
- 日時・件数など事実は正確に。未確定は「調査中」と明記。
- 社外向けは専門用語を避け、影響と対応と再発防止が伝わることを優先。
- 「担当者の操作ミス」で止めず、なぜそれを防げなかったか(仕組み)まで書く。
