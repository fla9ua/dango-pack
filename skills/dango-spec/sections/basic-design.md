# 設計書: 基本設計書(外部設計)

「外から見える形」を決める。ユーザー・他システムから見たインターフェース。
要件定義書の機能要件(FR-xx)を、画面・API・データに落とす。

## 原則
- 各設計項目を要件ID(FR-xx)に紐づけ、要件→設計の追跡性を保つ。
- 入力チェック・帳票・日付表示など、日本語特有の前提は `dango-review` の観点を
  設計段階で取り込む(後工程での手戻りを防ぐ)。
- 既存コードから起こす場合、実装を正として図・表を生成し、差異は「要確認」。
