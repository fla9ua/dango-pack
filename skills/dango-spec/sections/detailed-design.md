# 設計書: 詳細設計書(内部設計)

「中の作り」を決める。実装者がこれを見れば書ける粒度。
基本設計の各項目を、モジュール・処理・データ構造に落とす。

## 構成(雛形)

```
# 詳細設計書 — {{機能名}}

## 改訂履歴
| 版 | 日付 | 変更者 | 変更内容 |

## 1. モジュール構成

| モジュール | 責務 | 主な依存 |
|---|---|---|

## 2. 処理フロー
{{主要処理のシーケンス図/フローチャート(Mermaid)。分岐・ループを明示}}

## 3. データ構造
{{型定義・テーブル定義(物理)。カラム型・桁・NULL可否・既定値}}
- 日付の保持形式・タイムゾーン、文字コード(UTF-8)、正規化方針(NFKC)を明記

## 4. インターフェース詳細
{{関数/メソッドのシグネチャ、引数・戻り値・前提条件}}

## 5. エラー処理・例外

|---|---|---|---|

{{排他制御・整合性の取り方}}

## 7. テスト観点
{{単体テストで確認する条件。境界値・異常系。日本語特有(全角半角/和暦/IME)を含む}}
```

## 原則
- 実装者が迷わない粒度まで具体化する(が、コードのコピーにはしない)。
- 文字コード・日付・正規化など、日本語まわりの実装前提をここで固定する。
- テスト観点に日本語特有のケース(`dango-review` の5観点)を必ず1つ以上含める。
- 逆生成時は実装を正とし、設計意図が不明な箇所は「要確認」と残す。
