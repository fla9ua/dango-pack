# 観点: 禁則処理・改行・折り返し

日本語の組版ルール。CSSのデフォルトは欧文向けなので、日本語UIでは崩れやすい。

## 検出パターン

### 1. 行頭・行末禁則
- 行頭に来てはいけない: `、。，．）」』】〕!?ー…` など。
- 行末に来てはいけない: `（「『【〔` など。
- ブラウザの標準折り返しでこれらが行頭/行末に来ていないか目視+設定確認。

### 2. CSS の折り返し設定
- 日本語に不適切な指定:
  - `word-break: break-all`(英単語まで途中でぶつ切り。原則避ける)
  - `line-break` 未指定で約物が不自然に折り返す。
- 推奨の出発点:
  ```css
  word-break: normal;
  overflow-wrap: anywhere;   /* 長いURL等のはみ出し対策 */
  line-break: strict;        /* 日本語の禁則を厳格に */
  ```
- Grep 例: `rg 'word-break|line-break|overflow-wrap|white-space'`

### 3. 英単語・URL・数字の途中改行
- 半角英数の連続(URL、メール、長い英単語)がコンテナをはみ出す/不自然に
  割れる。`overflow-wrap: anywhere` か `<wbr>` の検討。

### 4. 文の途中で意味が切れる改行
- `<br>` で見た目を整えるための強制改行が、画面幅で不自然になっていないか。
- ボタン/見出しの「。」や助詞単独の行送りなど、読みづらい折り返し。

### 5. 行の高さ・字間
- `line-height` が日本語に対して詰まりすぎ(欧文基準の 1.2 等)。日本語本文は
  1.6〜1.8 程度が読みやすい、という観点で確認。

## 直し方の指針
- 本文ブロックに `line-break: strict; overflow-wrap: anywhere; word-break: normal;`
  を基本セットとして提案。
- `break-all` を見たら理由を確認し、英数字混在なら `anywhere` に置換を提案。
