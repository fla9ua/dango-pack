# 観点: 全角半角の混在

日本語プロダクトで最も頻出する地雷。表示・入力・データ保存のどこで揺れても
バグや検索漏れ、見た目の崩れになる。

## 検出パターン

### 1. 全角数字・全角英字
- 文字列リテラルやUIテキストに `０１２３４５６７８９` や `Ａ-Ｚ ａ-ｚ` が混入。
- 原則、数値・英字は半角。全角は固有名詞や意図的な装飾のみ許容。
- Grep 例: `rg '[０-９Ａ-Ｚａ-ｚ]' --glob '*.{ts,tsx,vue,html,json}'`

### 2. 半角カナ
- `ｱｲｳ` 等の半角カナは機種依存・文字化けの温床。原則 全角カナへ。
- Grep 例: `rg '[ｦ-ﾟ]'`

### 3. 全角スペース(U+3000)の混入
- コード・クラス名・フォーム入力値・正規表現に紛れ込みやすい。
- 入力フォームでは前後の全角スペースを trim しているか確認。
- Grep 例: `rg '　'`(全角スペースそのもの)

### 4. 句読点・記号の不統一
- `、。` と `，．` と `,.` が混在していないか(プロダクト内で1種類に統一)。
- 全角チルダ `〜`(U+301C)と波ダッシュ `～`(U+FF5E)の混同。
- 機種依存文字: 丸数字 `①②`、ローマ数字 `ⅠⅡ`、`㈱㍻` 等。データ保存やメール
  送信で化ける可能性を指摘。

### 5. 入力値の正規化(NFKC)
- ユーザー入力(氏名・住所・検索語)を保存/比較する前に正規化しているか。
- 例: `str.normalize('NFKC')` で全角英数→半角、半角カナ→全角に寄せる。
- 検索機能で「Ａpple」と「Apple」が別物になっていないか。

## 直し方の指針
- 表示テキスト: 半角に統一(全角が意図的ならコメントで明示を提案)。
- 入力値: 受け取り直後に `normalize('NFKC')` + `trim()`。
- 比較/検索キー: 正規化済みの値で突き合わせる。
