# 観点: インボイス制度(適格請求書等保存方式)

事業者向けに課金・請求書を発行するなら関わる。BtoB の SaaS や、事業者が経費にする
有料サービスで特に重要。BtoC オンリーなら優先度は下がる。

## 点検

- 領収書/請求書の発行機能があるか。あるなら適格請求書の要件を満たすか。
  - Grep 例: `rg -i '請求書|領収書|invoice|receipt|適格|登録番号|インボイス'`
- 登録番号(T+13桁)を表示する欄があるか(課税事業者で登録済みの場合)。

## 適格請求書に必要な記載事項(雛形)

```
1. 発行事業者の氏名・名称 および 登録番号(T0000000000000)
2. 取引年月日
3. 取引内容(軽減税率対象ならその旨)
4. 税率ごとに区分した合計額 と 適用税率
5. 税率ごとに区分した消費税額等
6. 交付を受ける事業者の氏名・名称
```

## よくある論点

- **自社が免税事業者の場合** — 登録番号を持たないなら適格請求書は発行できない。
  顧客(事業者)が仕入税額控除できない点をどう案内するか。**要・最新確認。**
- **少額特例・経過措置** — 期限付きの特例があるため、必ず最新の制度を確認する。
- **決済代行(Stripe等)経由** — 領収書の発行主体が誰になるかを整理する。

## 重大度
- BtoB課金で請求書発行が要件項目を欠く: 中〜高(顧客の控除に影響)
- BtoCのみ: 低(多くは任意。ただし領収書を出すなら整合を取る)
