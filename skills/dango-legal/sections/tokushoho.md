# 観点: 特定商取引法に基づく表記

有料の商品・サービスをネット販売する事業者には、特商法に基づく表記の掲示義務が
ある。個人開発の有料アプリ/SaaS/EC で最も多い「知らずに未対応」の代表格。

## 点検

- サイト/アプリに「特定商取引法に基づく表記」ページが存在するか。
  - Grep 例: `rg -i '特定商取引|特商法|tokushoho|legal/commerce'`
- フッターや決済導線から到達できるか(埋もれていないか)。
- 後述の必須項目が埋まっているか(空欄・プレースホルダのまま公開していないか)。

## 必須で記載する項目(雛形)

```
# 特定商取引法に基づく表記

販売事業者名: {{事業者名 / 個人なら氏名}}
運営統括責任者: {{氏名}}
所在地: {{住所}}  ※請求があれば遅滞なく開示する旨の記載で代替可の場合あり(要確認)
電話番号: {{電話番号}}  ※同上
メールアドレス: {{連絡先メール}}
販売価格: {{価格 / 価格ページへの導線}}(消費税の扱いを明記)
商品代金以外の必要料金: {{送料・手数料など}}
支払方法: {{クレジットカード等}}
支払時期: {{都度 / 月額など}}
引渡時期(役務提供時期): {{購入後すぐ / 〇営業日以内など}}
返品・キャンセル: {{デジタル商品の返品可否・条件}}
```

## 個人開発でよくある論点

- **住所・電話の公開** — 個人だと出したくない。一定要件下で「請求時に遅滞なく開示」
  とできる場合があるが、プラットフォーム(各ストア/決済)の規約や最新の運用に依存。
  **要・最新確認**として専門家確認を促す。
- **サブスク(定期課金)** — 価格・更新時期・解約方法を明確に。誤認させない導線が必要。
- **デジタル商品の返品** — 原則「返品不可」とするなら、その旨を明示しないと無効に
  なりうる。

## 重大度
- ページ自体が無い/決済しているのに未掲示: **高(公開前に必須)**
- 項目の一部欠落: 中
